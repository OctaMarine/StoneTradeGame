  Вот как можно хранить и управлять информацией о рецептах крафта, а также отправлять ее с бэкенда на фронтенд:

  1. Бэкенд (C# .NET 8, Clean Architecture)

  1.1. Модель данных для рецептов крафта


  В контексте Clean Architecture, вы определите эти модели в доменном слое.


   * `CraftingRecipe` (Рецепт крафта):
       * Id (int): Уникальный идентификатор рецепта.
       * ResultItemId (int): Идентификатор предмета, который получается в результате крафта.
       * ResultQuantity (int): Количество предметов, получаемых в результате крафта.
       * ChanceOfSuccess (double): Шанс успешного крафта (например, 0.95 для 95%).
       * RequiredItems (List<CraftingIngredient>): Список необходимых ингредиентов.
       * CraftingTimeSeconds (int, optional): Время, необходимое для крафта (если есть задержка).
       * CraftingType(int/Guid, optional): Идентификатор вида крафта(дробление, слияние).


   * `CraftingIngredient` (Ингредиент крафта):
       * ItemId (int/Guid): Идентификатор необходимого предмета.
       * Quantity (int): Количество этого предмета.

  1.2. Хранение и управление рецептами


  Для хранения рецептов крафта у вас есть несколько вариантов, но для гибкости и возможности изменять рецепты без перекомпиляции сервера, база данных
  является лучшим выбором.


   * База данных (предпочтительно):
       * DAL (Data Access Layer): Создайте сущности CraftingRecipe и CraftingIngredient в вашем StoneActionServer.DAL.Models (или аналогичном) и
         настройте их с помощью Entity Framework Core в AppDbContext.
       * Репозитории: Создайте ICraftingRepository в вашем StoneActionServer.DAL.Repositories и его реализацию (CraftingRepository), чтобы обрабатывать
         операции CRUD (создание, чтение, обновление, удаление) для рецептов крафта.
       * Конфигурации: Добавьте конфигурации для CraftingRecipe и CraftingIngredient в StoneActionServer.DAL.Configurations, как вы это делаете для
         других сущностей (например, InventoryConfiguration).


   * Альтернатива (для статических рецептов):
       * JSON/XML файлы: Если рецепты крафта очень статичны и не меняются часто, вы можете хранить их в JSON-файлах на сервере и загружать их в память
         при запуске приложения. Однако, это менее гибко, чем база данных.

  1.3. Сервисы для логики крафта


   * BusinessLogic Layer: Создайте ICraftingService в StoneActionServer.BusinessLogic.Services и его реализацию (CraftingService). Этот сервис будет
     отвечать за:
       * Получение списка всех рецептов крафта.
       * Проверку наличия ингредиентов у пользователя.
       * Выполнение операции крафта (уменьшение ингредиентов, добавление созданного предмета, расчет шанса успеха).


  1.4. API Endpoint для отправки рецептов на фронтенд


   * WebApi Layer: Создайте новый контроллер, например, CraftingController.cs в StoneActionServer.WebApi.Controllers.
   * Эндпоинт: Реализуйте GET-эндпоинт для получения всех рецептов крафта:



    1     // StoneActionServer.WebApi/Controllers/CraftingController.cs
    2     [ApiController]
    3     [Route("api/[controller]")]
    4     public class CraftingController : ControllerBase
    5     {
    6         private readonly ICraftingService _craftingService;
    7
    8         public CraftingController(ICraftingService craftingService)
    9         {
   10             _craftingService = craftingService;
   11         }
   12
   13         [HttpGet("recipes")]
   14         public async Task<ActionResult<IEnumerable<CraftingRecipeDto>>> GetCraftingRecipes()
   15         {
   16             var recipes = await _craftingService.GetAllCraftingRecipesAsync();
   17             // Возможно, вам понадобится маппинг из доменной модели в DTO
   18             var recipeDtos = recipes.Select(r => new CraftingRecipeDto
   19             {
   20                 Id = r.Id,
   21                 ResultItemId = r.ResultItemId,
   22                 ResultQuantity = r.ResultQuantity,
   23                 ChanceOfSuccess = r.ChanceOfSuccess,
   24                 RequiredItems = r.RequiredItems.Select(i => new CraftingIngredientDto { ItemId = i.ItemId, Quantity = i.Quantity }).ToList()
   25             });
   26             return Ok(recipeDtos);
   27         }
   28
   29         // Возможно, позже добавите POST-эндпоинт для выполнения крафта
   30         // [HttpPost("craft")]
   31         // public async Task<IActionResult> Craft([FromBody] CraftRequest request) { ... }
   32     }
   * DTOs: Создайте соответствующие DTO (Data Transfer Objects) в StoneActionServer.WebApi.DTO для CraftingRecipe и CraftingIngredient, чтобы избежать
     передачи всех полей доменной модели напрямую на фронтенд.

  2. Фронтенд (Next.js, React, TypeScript)

  2.1. Потребление данных о рецептах крафта


   * `lib/api.ts`: Расширьте ваш существующий lib/api.ts для включения функции получения рецептов крафта.



    1     // Frontend/src/lib/api.ts
    2     // Предполагая, что у вас уже есть базовый API-клиент или fetch-функция
    3
    4     export interface CraftingIngredient {
    5       itemId: string; // Или number, в зависимости от типа Id на бэкенде
    6       quantity: number;
    7     }
    8
    9     export interface CraftingRecipe {
   10       id: string;
   11       resultItemId: string;
   12       resultQuantity: number;
   13       chanceOfSuccess: number;
   14       requiredItems: CraftingIngredient[];
   15     }
   16
   17     export async function fetchCraftingRecipes(): Promise<CraftingRecipe[]> {
   18       const response = await fetch('/api/crafting/recipes'); // Используйте относительный путь, Next.js будет проксировать
   19       if (!response.ok) {
   20         throw new Error('Failed to fetch crafting recipes');
   21       }
   22       return response.json();
   23     }


   * `src/app/game/page.tsx` (или где угодно, где будет панель крафта):
       * В компоненте GameMenu/Panel.tsx или родительском компоненте, который рендерит эту панель, используйте useEffect (или библиотеку для управления
         состоянием и запросами, такую как React Query/SWR) для получения данных.



    1     // Пример в компоненте, который будет использовать рецепты
    2     import React, { useState, useEffect } from 'react';
    3     import { fetchCraftingRecipes, CraftingRecipe } from '@/lib/api';
    4
    5     const CraftingPanel: React.FC = () => {
    6       const [recipes, setRecipes] = useState<CraftingRecipe[]>([]);
    7       const [loading, setLoading] = useState(true);
    8       const [error, setError] = useState<string | null>(null);
    9
   10       useEffect(() => {
   11         const getRecipes = async () => {
   12           try {
   13             const data = await fetchCraftingRecipes();
   14             setRecipes(data);
   15           } catch (err) {
   16             setError('Failed to load crafting recipes.');
   17             console.error(err);
   18           } finally {
   19             setLoading(false);
   20           }
   21         };
   22         getRecipes();
   23       }, []);
   24
   25       if (loading) return <div>Загрузка рецептов...</div>;
   26       if (error) return <div>Ошибка: {error}</div>;
   27
   28       return (
   29         <div>
   30           <h2>Рецепты крафта</h2>
   31           {recipes.length === 0 ? (
   32             <p>Рецепты крафта пока отсутствуют.</p>
   33           ) : (
   34             <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
   35               {recipes.map((recipe) => (
   36                 <div key={recipe.id} className="border p-4 rounded shadow">
   37                   <h3>Крафт: {recipe.resultQuantity} x ИмяПредмета({recipe.resultItemId})</h3>
   38                   <p>Шанс успеха: {(recipe.chanceOfSuccess * 100).toFixed(0)}%</p>
   39                   <h4>Ингредиенты:</h4>
   40                   <ul>
   41                     {recipe.requiredItems.map((ingredient, index) => (
   42                       <li key={index}>
   43                         {ingredient.quantity} x ИмяИнгредиента({ingredient.itemId})
   44                       </li>
   45                     ))}
   46                   </ul>
   47                   <button className="mt-2 bg-blue-500 text-white px-4 py-2 rounded">Крафтить</button>
   48                 </div>
   49               ))}
   50             </div>
   51           )}
   52         </div>
   53       );
   54     };
   55
   56     export default CraftingPanel;
       * В GameMenu/Panel.tsx вы будете условно рендерить CraftingPanel при выборе вкладки "Крафт".

  2.2. Отображение рецептов в `GameMenu/Panel.tsx`

  Когда вы получите данные о рецептах, вам нужно будет отобразить их:


   1. Компонент CraftingPanel: Создайте отдельный React компонент (CraftingPanel.tsx) внутри src/components/features/GameMenu/ или рядом с ним. Этот
      компонент будет отвечать за отображение списка рецептов.
   2. Карточки рецептов: Для каждого CraftingRecipe создайте "карточку" или элемент списка, который будет четко отображать:
       * Название/иконку получаемого предмета и его количество.
       * Шанс успешного крафта.
       * Список необходимых ингредиентов (название/иконка и количество).
       * Кнопку "Крафтить".
   3. Управление состоянием:
       * Вам потребуется информация об инвентаре пользователя, чтобы определить, может ли он крафтить тот или иной предмет (например, можно ли выделить
         кнопку "Крафтить", если ингредиентов недостаточно). Эта информация должна быть получена из другого API-эндпоинта и доступна в родительском
         компоненте или через контекст/Redux/Zustand.
       * Когда пользователь нажимает кнопку "Крафтить", вы отправляете запрос на бэкенд (например, POST /api/crafting/craft) с ID рецепта. Бэкенд
         обрабатывает логику крафта и возвращает результат (успех/неудача, полученные предметы, обновленный инвентарь).
   4. Стилизация: Используйте Tailwind CSS для создания адаптивного и привлекательного макета для карточек рецептов и общей панели крафта.

  Резюме:


  Этот подход обеспечивает четкое разделение обязанностей между бэкендом и фронтендом, что соответствует принципам Clean Architecture. Бэкенд отвечает
  за хранение, управление и логику крафта, а фронтенд — за получение и отображение информации о рецептах, а также за взаимодействие с пользователем.
